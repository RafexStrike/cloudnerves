## Responsive Behavior

### Desktop (lg and above)
- Pending requests in 2-column grid layout
- Full width stats cards
- Modal centered on screen
- Full blocked students list

### Tablet (md and above)
- Pending requests in 2-column grid
- Stats in 4-column grid
- Modals centered

### Mobile (sm and below)
- Pending requests stack vertically (1 column)
- Stats stack vertically (1 column)
- Buttons stack when needed
- Modal responsive with proper padding

## Interactive States

### Buttons
- **Normal**: Full color, cursor pointer
- **Hover**: Enhanced shadow, scale up slightly
- **Active**: Pressed state with darker shade
- **Disabled**: Grayed out, cursor disabled, loading spinner

### Loading States
- **During Action**: Button shows spinner + "Processing..."
- **Modal**: Shows spinner during block confirmation
- **List**: Shows skeleton or spinner while loading

## Animations & Transitions

- **Card Hover**: `transition-all` with `hover:shadow-xl`
- **Button Click**: Smooth state transition
- **Modal Appearance**: Fade in with backdrop
- **Message Alerts**: Slide in from top, auto-dismiss in 2-3 seconds
- **Tab Refresh**: Smooth data update without page reload
